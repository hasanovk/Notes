# Written by KFH: karshi.hasanov@utoronto.ca # Date: May 10, 2010 # Last Modified: August 6, 2017 # Description: Installing the OpenSSL server and generating the CA’s Installing the OpenSSL Server: => sudo portsnap fetch update # We want to install the OpenSSL Server under the “/usr/local/openssl” folder. # Edit the “/etc/make.conf” file and add NO_OPENSSL = yes => sudo vi /etc/make.conf # Add this : NO_OPENSSL = yes and this DEFAULT_VERSIONS+=ssl=openssl # The port location => cd /usr/ports/security/openssl # Override the default OpenSSL install => sudo make -D OPENSSL_OVERWRITE_BASE install clean # This will install the OpenSSL under "/usr/local/openssl" # Remove 'old' binary : => sudo mv /usr/bin/openssl /usr/bin/openssl_orig # Make the links to new location: => sudo ln -s /usr/local/bin/openssl /usr/bin/openssl => sudo ln -s /usr/local/openssl/openssl.cnf /etc/ssl/openssl.cnf Creating a SSL Certificate of Authority (CA) If you want to use a SSL certificate for things such as Apache
